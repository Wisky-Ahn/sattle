### 프로젝트 개요

- **핵심 아이디어**: 개발 교육용 환경 세팅을 자동화하는 서비스
- **문제 인식**: 수업 중 환경 세팅으로 인한 시간 소요와 학생들의 어려움 해결
- **타겟**: 프레임워크 수업 (Python, Java, HTML, CSS, JS, Linux, 서버 관리, 하네스 개발, DB 등)
- **차별점**: 단순 언어 설치가 아닌 프레임워크 환경 세팅에 집중 (예: Android Studio, Spring 등)

### 핵심 기능 및 시나리오

**서비스 플로우**   :

1. 강사가 강의에 필요한 환경 세팅 명세를 .md 파일로 제공
2. 사용 허가/결제 처리
3. 설치 프로그램(exe/dmg) 제공
4. 학생에게 배포
5. 학생이 실행하면 자동으로 환경 세팅 완료
6. 예제 테스트 코드 실행 후 설치 완료 여부 판단
7. 강사에게 설치 피드백 제공

**구현 방식**  :

- 학생 컴퓨터에 AI 하네스 설치
- 관리자 권한으로 실행
- 비밀번호 입력 후 자동 설치 진행
- 설치 완료 후 하네스 자동 삭제

### 기술 스택 및 구현 방향

**핵심 기술**:

- 하네스(Harness) 개발 프레임워크 사용
- AI 에이전트를 활용한 자동 환경 구축
- MCP(Model Context Protocol) 통합
- OpenClaw 또는 나노클로우(Nanoclaw) 고려

**프레임워크 설치 예시**  :

- Spring 프로젝트: Kotlin/Groovy/Java 선택, Spring Boot 버전, 의존성(Lombok, Spring Web 등) 설정
- 버전 및 의존성 자동 체크

**데이터베이스 및 인증**:

- Supabase 활용
- 결제 시스템: Polar 사용 (한국 결제 지원)
- OAuth 인증 구현 예정

### 우선순위 및 범위

**1단계 (POC/MVP 범위)**:

1. 로그인 기능 
2. 결제 연동
3. 강의 명세서 기반 설치 파일 제공 및 테스트 

**핵심 구현 우선순위**  :

- 기능 구현 (Mac 버전) 최우선
- 프레임워크 3개 정도 예제 구현
- 실행 → 클릭 → 환경 세팅 완료까지의 사용자 경험 완성
- 로그인, 모니터링 등은 향후 확장 기능으로 시연

**강조 사항**  :

- "마법같이" 작동하는 경험 제공 필요
- 어떤 PC에서든 한 번의 클릭으로 환경 구축 완료
- 빌드가 제대로 되는지 자동 확인

### 기술적 고려사항

**환경 설정 복잡성**  :

- Mac의 기본 Python 버전과 강의 요구 버전 충돌 문제
- 네이티브 설치, UV, brew 등 다양한 설치 경로로 인한 경로 충돌
- 에러 처리가 핵심 과제

**보안 및 권한**  :

- 관리자 권한 필요 (sudo 명령어 사용)
- 필수 동의 절차를 통한 사용자 동의 확보
- 비밀번호 서버 저장 (유료 서비스로 정당화)

**인터페이스 단순화**  :

- 강사가 복잡한 설정 불필요
- 주요 프레임워크별로 하드코딩된 설정값 제공
- 버전 선택 옵션은 있되 기본값 제공
