# 설치 검증 및 품질 테스트

환경 세팅 자동화 서비스의 품질을 검증하는 스킬. 핵심은 "존재 확인"이 아닌 "경계면 교차 비교"이다.

## 검증 원칙

단순히 "파일이 있는가"가 아니라 "컴포넌트 A의 출력이 컴포넌트 B의 입력과 정확히 일치하는가"를 검증한다.

## 1. 경계면 검증 포인트

이 서비스에서 버그가 발생하기 쉬운 경계면:

| 경계면 | A 측 | B 측 | 검증 방법 |
|--------|------|------|----------|
| 명세→파서 | .md 원본 | 파서 JSON 출력 | 명세의 모든 항목이 JSON에 반영되었는지 교차 비교 |
| 파서→스크립트 | 설치 계획 JSON | 생성된 셸 스크립트 | JSON의 각 의존성이 스크립트에 install 명령으로 존재하는지 |
| 스크립트→OS | 스크립트 기대 환경 | 실제 OS 상태 | 스크립트가 가정하는 경로/명령이 OS에 존재하는지 |
| API→UI | API 응답 shape | UI가 기대하는 데이터 shape | API 응답 필드와 UI 컴포넌트 props 매칭 |
| 설치→검증 | 설치된 바이너리 | 검증 스크립트 기대값 | which/version 출력과 기대값 일치 |
| 결제→접근권한 | 결제 상태 | 다운로드 가능 여부 | 결제 완료 후 다운로드 링크 활성화 확인 |

## 2. 테스트 유형

### 단위 테스트
- 명세 파서: 다양한 .md 포맷 입력 → 올바른 JSON 출력
- 스크립트 생성: 설치 계획 JSON → 유효한 셸 스크립트
- API 핸들러: 요청/응답 형식 검증

### 통합 테스트
- 명세 업로드 → 파싱 → 스크립트 생성 → 다운로드 전체 플로우
- 결제 → 접근권한 부여 → 설치 시작 전체 플로우
- 설치 시작 → 상태 업데이트 → 강사 대시보드 반영

### 에러 시나리오 테스트
- 잘못된 명세 포맷 (필수 필드 누락, 잘못된 버전)
- 이미 설치된 프레임워크 (충돌 감지)
- 네트워크 단절 중 설치
- 디스크 공간 부족
- sudo 권한 거부
- 지원하지 않는 프레임워크/버전

## 3. 점진적 QA 체크리스트

각 모듈 완성 직후 수행하는 검증:

### 명세 파서 완성 후
- [ ] 3개 프레임워크(Spring, Django, React) 명세 파싱 성공
- [ ] 잘못된 명세 입력 시 명확한 에러 메시지
- [ ] JSON 출력 스키마가 스크립트 생성기의 입력 스키마와 일치

### 스크립트 생성기 완성 후
- [ ] 생성된 스크립트가 bash -n으로 문법 검증 통과
- [ ] Intel Mac과 Apple Silicon 분기 로직 존재
- [ ] 각 단계의 에러 핸들링(set -e, trap) 존재
- [ ] 멱등성: 2회 연속 실행해도 에러 없음

### API 완성 후
- [ ] 모든 엔드포인트의 요청/응답 형식이 문서와 일치
- [ ] 인증 없이 접근 시 401 반환
- [ ] RLS 정책: 다른 사용자의 데이터 접근 불가

### UI 완성 후
- [ ] API 응답의 모든 필드를 UI가 올바르게 표시
- [ ] 설치 상태 변경 시 실시간 UI 업데이트
- [ ] 에러 상태에서 사용자에게 이해 가능한 메시지 표시

## 4. 테스트 결과 보고 형식

```markdown
# QA 검증 보고서

## 요약
- 전체 테스트: N개
- 통과: N개
- 실패: N개
- 통과율: N%

## 경계면 검증 결과
| 경계면 | 상태 | 발견된 문제 |
|--------|------|-----------|
| 명세→파서 | PASS/FAIL | 상세 |

## 실패 상세
### [실패 테스트명]
- 기대 결과:
- 실제 결과:
- 재현 방법:
- 영향 범위:
- 권장 수정:
```

## 5. 자동 검증 스크립트 패턴

설치 완료 검증에 사용하는 공통 스크립트 패턴:

```bash
#!/bin/bash
# verify_install.sh — 설치 검증 공통 패턴

RESULTS=()

check() {
  local name="$1" cmd="$2" expected="$3"
  local actual
  actual=$(eval "$cmd" 2>/dev/null) || { RESULTS+=("FAIL|$name|command failed"); return; }
  if [[ "$actual" == *"$expected"* ]]; then
    RESULTS+=("PASS|$name|$actual")
  else
    RESULTS+=("FAIL|$name|expected=$expected actual=$actual")
  fi
}

# 사용 예시
# check "java_version" "java -version 2>&1 | head -1" "17"
# check "gradle_build" "cd /path && gradle build" "BUILD SUCCESSFUL"

# 결과 출력
for r in "${RESULTS[@]}"; do echo "$r"; done
```
